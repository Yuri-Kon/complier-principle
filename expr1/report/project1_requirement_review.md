# Project 1 Requirement Review

基于 `../实验教程-Project_1.pdf` 第 2.1 节与当前代码状态整理。目标是先对齐实验要求，再判断当前实现范围与偏差。

## 1. 需求摘要（按实验手册）

### 1.1 必做内容
- 输入：程序接收一个源文件路径作为命令行参数。
- 输出：若存在词法错误或语法错误，只输出错误信息，格式为 `Error type [A/B] at Line [n]: ...`，且不要输出语法树。
- 正确输入：按先序遍历打印语法树；非终结符打印名称和首行号，词法单元打印名称，`ID/TYPE/INT/FLOAT` 额外打印词素或数值。

### 1.2 选做内容
- 识别八进制、十六进制整数。
- 识别指数形式浮点数。
- 识别 `//` 与 `/* ... */` 注释，并对非法注释报错。

### 1.3 工程要求
- 使用 Flex、Bison、C。
- 代码结构要利于后续实验继续扩展。

## 3. 已实现的核心模块

### 3.1 词法分析
- 关键字、标识符、整数、浮点数、运算符、分隔符都已覆盖。
- `SKIPLINE` 状态用于在词法错误后跳过同一行后续内容，避免重复报错。
- 注释被过滤，不进入语法分析。

### 3.2 语法分析与构树
- 已实现的主要非终结符包括 `Program`、`ExtDefList`、`Specifier`、`StructSpecifier`、`CompSt`、`DefList`、`StmtList`、`Exp`。
- 已支持的典型结构：全局变量定义、无参/带参函数、局部变量定义、数组定义、结构体定义/引用、函数调用、赋值、算术运算、逻辑运算、`if/else`、`while`、`return`。
- 多叉树结构与打印逻辑已经独立到 `tree.h/tree.c`，这点符合后续实验扩展需要。

## 4. 当前未覆盖或偏离规范的部分

### 4.1 仍需注意的边界点
- 当前实现已经覆盖实验一常用标准文法，但错误恢复仍以“常见同步点”为主，不保证所有非法输入都得到最细粒度说明。
- 部分非常规错误样例仍可能输出通用 `syntax error`，这是可接受的，但不如定向提示稳定。
- 注释处理按实验一需求足够使用，但未尝试支持真正的嵌套块注释；相关输入会转化为后续语法错误。

## 5. 建议的下一步

1. 若继续以评分为优先，建议把当前测试集对应的期望输出整理成对照表，固定回归基线。
2. 若开始为实验二做准备，建议在语法树结点中增加可扩展字段，例如结点类别枚举、类型信息槽位或源码位置结构。
3. 若想进一步提升错误质量，可继续补 `Missing ")"`、`Missing ";"`、`Missing "]"` 之外的定向提示，但要避免为了报错而破坏标准文法。
